## 재귀함수

함수 내부에서 자기자신을 다시 호출해 작업을 수행함

분할 정복 알고리즘 

- top down
재귀에서는 :  n → n-1 이렇게 내려감
최종 계산은 재귀가 끝나고 위로 돌아오면서(return) 이루어짐
재귀 + 메모이제이션 활용

    
    ```cpp
    def top_down_fact(n):
        if n == 1: 
            return 1
        return n * top_down_fact(n - 1) # 위에서 아래로 호출, 계산은 돌아오면서!
    
    ```
    
- bottom up
재귀에서는  : n-1 → n 이렇게 올라감
파라미터에 계산결과 저장할 매개변수 추가.
계산은 재귀가 내려가는 과정에서(매개변수 전달) 이미 완료됨
반복문 + 타뷸레이션이 해당

    
    ```cpp
    def bottom_up_fact(n, current=1, accum=1):
        if current > n: 
            return accum
        # 아래에서 위로 올라가며 매개변수(accum)에 미리 계산 결과를 저장!
        return bottom_up_fact(n, current + 1, accum * current) 
    
    ```
    

두 가지 방식이 있다

분할 정복 알고리즘에는

- 분할
예 : 이진탐색에서 배열 반으로 갈라 x가 중앙에 위치한 항목보다 작으면 왼쪽 배열 반쪽 선택
- 정복
예 : 선택된 반쪽 배열에서 x찾음

재귀에는

- 재귀 종료 조건 base case
- 자기 자신 콜 recursive case

문제 키워드에 재귀함수 뿐만 아니라 관계기반설계도 포함된 경우를 찾을 수 있는데,

코드업에서 말하는 관계기반설계

: 재귀함수나 수학적 점화식(수열)을 활용해 복잡한 문제를 작은 단위의 관계로 쪼개(분할) 해결하는 알고리즘 설계 방식을 말함.

top down과 bottom up형식이 모두 포함됨

이때

- top down
재귀 + 메모이제이션
큰 문제 풀기 위해 작은 문제 호출하는 방식
코드업에서는 기초적인 재귀함수 문제들에 해당 (시간/메모리 초과, 스택 오버플로우의 위험)
- bottom up
반복 + 타뷸레이션
작은 문제부터 차례로 계산해 큰 문제 해결
top down 형식에 비해 메모리 사용량과 시간 면에서 더 효율적이다
주로 큰 문제나 동적 계획법 문제에 이용됨


# 파스칼 삼각형이란?
1. 가장 첫 번째 줄의 중앙에 숫자 1 적음
2. 그 다음 줄부터는 바로 위 줄에 인접한 두 숫자의 합을 아래에 적음
3. 삼각형의 양쪽 바깥쪽 테두리는 항상 1로 채워짐

행(n)
        0:                     1
        1:                   1   1
        2:                 1   2   1
        3:               1   3   3   1
        4:             1   4   6   4   1
        5:           1   5  10  10   5   1

- 파스칼의 삼각형의 각 숫자는 조합 기호인 $_nC_r$ 의 값과 정확히 일치
- 조합의 성질을 이용해 알고리즘 구현 시 사용되는 수학적 정의는 $$\binom{n}{r} = \binom{n-1}{r-1} + \binom{n-1}{r}$$
- (내 칸의 값은 바로 왼쪽 위의 값과 오른쪽 위의 값을 더한 것과 같다)
